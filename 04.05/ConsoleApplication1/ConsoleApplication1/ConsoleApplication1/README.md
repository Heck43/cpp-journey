# ДЗ 04.05 — Класс «Дробь»

> простые дроби с автосокращением, арифметикой и цепочками вызовов через `this`

---

## Условие

Создать класс `Fraction` для работы с простыми дробями:
- поля: числитель и знаменатель
- конструктор с параметрами (список инициализаторов)
- конструктор по умолчанию через делегирование
- сложение / вычитание / умножение — с дробью и с целым числом
- цепочки вызовов через `this`
- сокращение дроби в конструкторе

---

## Реализация

**Сокращение** происходит автоматически в конструкторе через приватный метод `reduce()`.
Внутри — алгоритм Евклида для нахождения НОД:

```cpp
while (b != 0) {
    int temp = b;
    b = a % b;
    a = temp;
}
```

Знак всегда переносится в числитель — знаменатель хранится положительным.
Поэтому `Fraction(1, -2)` и `Fraction(-1, 2)` дадут одинаковый результат.

**Конструктор по умолчанию** делегирует в основной — не дублирует `reduce()`:

```cpp
Fraction() : Fraction(0, 1) {}  // 0/1 = ноль
```

**Перегрузка методов** — `add`, `sub`, `mul` принимают и `Fraction`, и `int`.
C++ выбирает нужный вариант по типу аргумента.

**Цепочки вызовов** работают потому что каждый метод возвращает `Fraction&` (ссылку на себя):

```cpp
return *this;  // возвращаем сам объект, а не копию
```

Это позволяет писать в одну строку:

```cpp
Fraction(1, 6).add(Fraction(1, 6)).add(Fraction(1, 6)).print();  // 1/2
```

---

## От себя

самое интересное — алгоритм Евклида, раньше писал его только для обычных чисел, а тут он внутри класса как приватный метод. и `reduce()` вызывается в конструкторе — дробь гарантированно приходит в сокращённом виде сразу при создании, не надо помнить вызывать вручную.

`return *this` поначалу выглядит странно, но идея простая: возвращаем ссылку на текущий объект, чтобы на нём сразу можно было снова вызвать метод <3
