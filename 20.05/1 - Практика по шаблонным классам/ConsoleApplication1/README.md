## Задание 1.1 — Класс Fraction (дробь)

### Условие
Реализовать класс `Fraction` для работы с дробями:
- хранение числителя и знаменателя
- перегрузка арифметических операторов: `+`, `-`, `*`, `/`
- перегрузка операторов сравнения: `==`, `!=`, `<`, `>`
- вывод через `<<`

### Реализация
Сделал класс с `private` полями `numerator` и `denominator`. Каждый арифметический оператор создаёт новую дробь по формуле. Операторы сравнения сравнивают через перемножение знаменателей. Вывод через перегруженный `<<`.

### От себя
перегрузка операторов — это магия :3 теперь можно писать `a + b` вместо `a.add(b)`, и код читается как математика

## Задание 1.2 — Класс Matrix (матрица)

### Условие
Реализовать класс `Matrix` с:
- динамическим выделением памяти
- перегрузкой `+`, `-`, `*` (матрица на матрицу и на скаляр)
- транспонированием через `!`
- конструктором копирования и оператором присваивания
- сравнением `==`, `!=`

### Реализация
Память выделяется в конструкторе через `new`, освобождается в деструкторе. Умножение матриц реализовано через тройной цикл. Оператор `!` создаёт новую матрицу с переставленными строками и столбцами.

### От себя
самое сложное было — конструктор копирования. без него всё ломается при присваивании, потому что два объекта начинают делить один и тот же кусок памяти

## Задание 2.1 — Класс Shape (фигуры)

### Условие
Создать иерархию фигур:
- абстрактный класс `Shape` с виртуальными методами `area()` и `perimeter()`
- статический счётчик фигур `getFigureCount()`
- производные классы: `Circle`, `Rectangle`, `Triangle`
- деструктор уменьшает счётчик

### Реализация
`Shape` — абстрактный класс с чистыми виртуальными функциями. В каждом производном классе своя формула площади и периметра. Статическое поле в базовом классе считает количество живых объектов.

### От себя
понял смысл полиморфизма — массив указателей `Shape*` может хранить любые фигуры, и `shapes[i]->area()` вызовет правильный метод для каждой. это реально круто <3

## Задание 2.2 — Класс Employees (сотрудники)

### Условие
Иерархия сотрудников:
- базовый класс `Employees`
- производные: `Manager`, `Developer`, `Intern`
- класс `Department` с добавлением, удалением, сортировкой и поиском

### Реализация
Каждый тип сотрудника переопределяет `getinfo()` и `getSalary()`. Департамент хранит список через `vector<Employees*>`. Сортировка через `std::sort` с лямбда-функцией по зарплате.

### От себя
самое интересное — поиск максимальной зарплаты через указатель. надо было осторожно с `nullptr` чтобы не упасть :3
