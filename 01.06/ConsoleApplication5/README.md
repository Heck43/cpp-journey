# Домашние животные — виртуальные функции

> иерархия питомцев: каждый издаёт свой звук и знает что он такое

---

## Условие

Создать базовый класс `Pet` (домашнее животное) и производные классы `Dog`, `Cat`, `Parrot`, `Hamster`. Через конструктор устанавливается имя. Реализовать для каждого:
- `Sound()` — издаёт звук
- `Show()` — выводит имя
- `Type()` — выводит название подвида

---

## Реализация

**Pet** — абстрактный базовый класс. `Sound()` и `Type()` — чисто виртуальные (`= 0`). `Show()` уже реализован в базовом классе, так как логика одинакова для всех — просто вывести имя.

Каждый производный класс переопределяет `Sound()` и `Type()` через `override`:
- `Dog` — "Гав-гав!"
- `Cat` — "Мяу-мяу!"
- `Parrot` — "Кра-кра! Попка дурак!"
- `Hamster` — "Пи-пи-пи!"

В `main()` все объекты хранятся в `vector<Pet*>` и вызываются через указатели — полиморфизм в деле. После работы память освобождается через `delete`.

---

## От себя

интересно что `Show()` не виртуальная в смысле "чистая" — она уже есть в базовом классе, потому что имя есть у всех. а вот `Sound()` и `Type()` без реализации в базовом — нельзя создать просто `Pet` без конкретного вида :3
