# Служащие — абстрактный класс и консольный интерфейс

> President, Manager, Worker — каждый сам знает как представиться

---

## Условие

Создать абстрактный базовый класс `Employer` с чисто виртуальной функцией `Print()`. Производные классы: `President`, `Manager`, `Worker`. Переопределить `Print()` для каждого типа с выводом соответствующей информации. Реализовать консольное меню для работы с проектом.

---

## Реализация

**Employer** — абстрактный класс. Хранит имя. `Print()` — чисто виртуальная.

**President** — хранит компанию и зарплату. В `Print()` выводит всё с заголовком `[Президент]`.

**Manager** — хранит отдел, зарплату и размер команды.

**Worker** — хранит должность, зарплату и стаж.

В `main()` создаётся `vector<Employer*>` с несколькими сотрудниками разных типов. Консольное меню позволяет:
- показать всех служащих (полиморфный `Print()`)
- отфильтровать по типу через `dynamic_cast`

Ввод защищён от некорректных данных через `cin.fail()` + `cin.clear()`.

---

## От себя

`dynamic_cast` — первый раз использовал по-настоящему. он возвращает `nullptr` если объект не того типа — удобно для фильтрации по типу без отдельного поля-флага.

меню с `do...while` и защитой ввода стало стандартным шаблоном — теперь пишу его автоматически :3
