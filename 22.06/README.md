## Реализация

**`Date`** хранит день/месяц/год и умеет вычислять разницу в днях через алгоритм порядкового числа — это позволяет точно фильтровать «последние 7 дней» независимо от переходов между месяцами и годами.

**Иерархия счетов** — наследование от абстрактного `Account`:
- `Wallet` и `DebitCard` — нельзя уйти в минус
- `CreditCard` переопределяет `withdraw()` — разрешает отрицательный баланс до кредитного лимита

```cpp
// CreditCard::withdraw — отличается от базового
if (balance - amount < -creditLimit) { /* ошибка */ }
balance -= amount;  // разрешаем минус
```

**Агрегация по категориям** для топ-3 — используем `std::map<string, double>`:
```cpp
std::map<std::string, double> catTotals;
for (const auto& t : list) catTotals[t.category] += t.amount;
// потом переносим в вектор пар и сортируем лямбдой
```

**Сохранение в файл** — `std::ofstream` с флагом `ios::app` (дописывает в конец, не перезаписывает). Файлы называются автоматически: `report_day_23.06.2026.txt`, `report_week.txt`, `report_month_06_2026.txt`.

**При запуске** сразу добавляются три тестовых счёта и несколько затрат за последние дни — чтобы можно было сразу посмотреть отчёты не вводя всё вручную.

## От себя

самое интересное — агрегация по категориям. сначала хотела сделать ручной подсчёт через вектор, но `std::map` решает это в три строки: ключ = категория, значение = сумма, и `+=` сразу накапливает. потом просто переносим в `vector<pair<string, double>>` и сортируем лямбдой.

дата с разностью дней тоже не очевидная штука — если просто сравнивать `day - other.day`, переход через начало месяца сломает всё. алгоритм порядкового числа переводит дату в одно целое и тогда разность — обычное вычитание <3
