## Задача 1.1 — Fraction

Класс дроби с перегруженными операторами — можно писать `a + b`, `a < b`, `cout << a`.

**`friend operator<<`** — оператор вывода объявлен `friend`, чтобы он мог видеть `private`-поля `numerator` и `denominator`:

```cpp
friend std::ostream& operator<<(std::ostream& os, const Fraction& f) {
    return os << f.numerator << "/" << f.denominator;
}
```

Деление на ноль бросает исключение — поймать можно через `try/catch`:

```cpp
try {
    Fraction bad = a / Fraction(0, 1);
} catch (const std::invalid_argument& e) {
    std::cout << e.what();  // "Деление на ноль!"
}
```

## Задача 1.2 — Vector3D

Все ключевые операции над трёхмерным вектором.

**Постфиксный инкремент** отличается от префиксного фиктивным параметром `int` — компилятор по нему понимает что это постфикс:

```cpp
Vector3D& operator++()    { x++; y++; z++; return *this; } // префикс: ++v
Vector3D  operator++(int) { Vector3D old(*this); ++(*this); return old; } // постфикс: v++
```

Постфикс сохраняет копию *до* увеличения и возвращает её — поэтому `v++` возвращает старое значение.

**Скалярное произведение** через `*` между двумя векторами: `a * b = ax*bx + ay*by + az*bz`.
Умножение вектора на число тоже `*`, но принимает `double` — C++ выбирает нужный вариант по типу аргумента.

## Задача 1.3 — Matrix\<T\>

Шаблонный класс — `T` это любой тип, обычно `int` или `double`.

**Правило трёх** обязательно из-за динамической памяти (`T** data`):

```cpp
Matrix(const Matrix& o);        // конструктор копирования
Matrix& operator=(const Matrix& o); // оператор присваивания
~Matrix();                       // деструктор
```

В операторе присваивания защита от самоприсваивания `a = a`:

```cpp
if (this == &o) return *this;  // без этого удалили бы свои же данные
```

**`operator!`** — транспонирование: строки и столбцы меняются местами, `matrix(i,j)` становится `result(j,i)`.

**`operator()`** вместо `[][]` — для двумерного доступа перегружают `()`, потому что `[][]` не перегружается напрямую:

```cpp
matrix(1, 2) = 5;  // устанавливаем элемент строки 1, столбца 2
```

## Задача 2.1 — Фигуры

Абстрактный базовый класс `Shape` с чистыми виртуальными методами — создать `Shape` напрямую нельзя.

```cpp
virtual double area() const = 0;  // = 0 — чисто виртуальный, обязателен в наследниках
```

Полиморфизм: массив указателей `Shape*` хранит разные фигуры, а `->area()` вызывает правильный метод для каждой:

```cpp
std::vector<Shape*> shapes = { new Circle(5), new Rectangle(3,4), new Triangle(3,4,5) };
for (const Shape* s : shapes) s->print();  // у каждого своя формула
```

Виртуальный деструктор `virtual ~Shape()` обязателен — без него `delete` через `Shape*` не вызвал бы деструктор дочернего класса.

Треугольник считает площадь по **формуле Герона**: `S = sqrt(s*(s-a)*(s-b)*(s-c))`, где `s = периметр/2`.

## Задача 2.2 — Сотрудники

Три типа сотрудников — у каждого своя формула зарплаты через `override`:

```cpp
// Manager:   базовая + бонус * 100 * подчинённых
// Developer: базовая + опыт * 5000
// Intern:    базовая * 0.70
```

`Department` хранит `vector<Employee*>` — указатели на базовый класс.
Это и есть полиморфизм: `calculateSalary()` вызывается у нужного типа автоматически.

**Сортировка** через лямбду — анонимная функция прямо внутри вызова `std::sort`:

```cpp
std::sort(employees.begin(), employees.end(),
    [](const Employee* a, const Employee* b) {
        return a->getSalary() < b->getSalary();
    });
```

В деструкторе `Department` удаляет каждый объект через `delete` — если забыть, утечка памяти.
