# Консольная RPG — пошаговый бой

> финальное задание модуля: игрок против гоблина, оружие, броня, зелья и боевая система

---

## Условие

Реализовать консольную пошаговую RPG с использованием ООП:

- иерархия персонажей: базовый класс `Character`, производные `Player` и `Enemy`
- иерархия предметов: базовый класс `Item`, производные `Weapon`, `Armor`, `Potion`
- класс `BattleSystem` управляет ходами и проверяет победу / поражение
- игрок сам выбирает действие каждый ход: атаковать, выпить зелье, осмотреть снаряжение

---

## Структура классов

```
Character  (базовый: имя, HP, атака, защита)
├── Player     — добавляет оружие, броню, зелья
└── Enemy      — добавляет тип врага

Item  (базовый: название, описание)
├── Weapon     — бонус к атаке
├── Armor      — бонус к защите
└── Potion     — лечение, одноразовое использование

BattleSystem   — игровой цикл, ходы, результат боя
```

---

## Реализация

**Character** — хранит имя, текущее и максимальное HP, базовую атаку и защиту. `takeDamage()` вычитает урон с учётом защиты. Методы `getAttackPower()` и `getDefenseValue()` — виртуальные, переопределяются в `Player`.

**Player** — экипирует оружие и броню через `new`, удаляет в деструкторе. Итоговая атака = базовая + бонус оружия, защита = базовая + бонус брони. Зелья хранятся в `vector<Potion*>`, `usePotion()` помечает зелье как использованное.

**BattleSystem** — главный цикл `while (player.isAlive() && enemy.isAlive())`. Ход игрока — меню выбора действия с валидацией ввода. Если выбрано "осмотреть снаряжение" — ход не засчитывается, меню показывается снова. После хода игрока — автоатака врага.

**Управление памятью** — все предметы создаются через `new` в `main.cpp`, передаются в `Player`. Деструктор `Player` удаляет всё через `delete`.
