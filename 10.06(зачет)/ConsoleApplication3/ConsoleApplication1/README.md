# Билет №4 Система покупки товара

> покупатель выбирает товары из каталога, а программа следит за его балансом

---

## Задание

Нужно реализовать систему магазина с помощью ООП.

Есть базовый класс товара и минимум три разных класса-наследника (например, электроника, одежда, еда — у каждого свои уникальные поля). Также нужен класс покупателя с полем баланса. Реализовать саму покупку: проверку баланса, списание денег и сохранение купленных товаров. При этом учитывать **клиентоориентированность** — пользователь должен понимать что происходит, получать понятные сообщения и сам выбирать что купить.

---

## Реализация

Иерархия классов:

```
product          ← базовый класс (имя, цена, id, displayInfo)
├── electronics  ← добавляет поле: модель
├── clothes      ← добавляет поле: размер
└── food         ← добавляет поле: срок годности

Human            ← покупатель (имя, баланс, корзина)
```

---

### class product — базовый класс товара

Хранит общие данные любого товара: название, цену и уникальный ID. Все три поля закрыты (`private`) — снаружи их можно только прочитать через геттеры.

```cpp
class product {
protected:
    std::string name;  // название товара
    double price;      // цена
    int id;            // уникальный идентификатор
public:
    std::string getName() const;   // вернуть имя
    double getPrice() const;       // вернуть цену
    int getId() const;             // вернуть id

    virtual void displayInfo() const;  // вывод информации — виртуальный!
};
```

`displayInfo()` объявлен как `virtual` — это значит, что каждый наследник может переопределить его по-своему. Именно это и есть полиморфизм.

---

### class electronics, clothes, food — наследники

Каждый добавляет своё уникальное поле и переопределяет `displayInfo()`:

```cpp
class electronics : public product {
    std::string model;       // модель устройства
    void displayInfo() const override;  // выводит базовый вывод + модель
};

class clothes : public product {
    std::string size;        // размер одежды
    void displayInfo() const override;  // выводит базовый вывод + размер
};

class food : public product {
    std::string expirationDate;  // срок годности
    void displayInfo() const override;  // выводит базовый вывод + срок
};
```

В каждом `displayInfo()` сначала вызывается `product::displayInfo()` для вывода общих данных, а потом дописывается специфичная строчка:

```cpp
void electronics::displayInfo() const {
    product::displayInfo();  // [ID: 1] Смартфон Samsung — 35990 руб.
    std::cout << "     Тип: Электроника | Модель: " << model << std::endl;
}
```

---

### class Human — покупатель

Хранит имя, баланс и **корзину** — вектор из указателей на купленные товары.

```cpp
class Human {
    std::string name;
    double balance;
    std::vector<product*> cart;  // корзина покупателя
public:
    bool buyproduct(std::vector<product*> items);  // купить товар
    void showCart() const;                          // показать корзину
};
```

**`buyproduct`** — главный метод. Считает суммарную стоимость переданных товаров, сравнивает с балансом. Если денег хватает — списывает и добавляет товар в `this->cart`. Если нет — выводит сообщение с нехваткой.

```cpp
if (balance >= totalCost) {
    balance -= totalCost;
    cart.push_back(item);  // сохраняем покупку в корзину
    std::cout << "Куплено: " << item->getName() << " за " << item->getPrice() << " руб.";
} else {
    std::cout << "Недостаточно средств! Нужно: " << totalCost << " руб., есть: " << balance;
}
```

**`showCart`** — проходит по `cart` и вызывает `displayInfo()` у каждого купленного товара, затем выводит итоговую сумму и остаток баланса.

### main — интерактивный интерфейс

В `main` создаётся витрина (`shopShowcase`) с шестью товарами трёх разных классов, и один покупатель с балансом 15 000 руб.

Дальше — цикл: показываем каталог, читаем ID от пользователя, ищем товар, вызываем `buyproduct`. При вводе 0 — выходим и показываем итоговую корзину:

```
========== Каталог товаров ==========
  [ID: 1] Смартфон Samsung — 35990 руб.
     Тип: Электроника | Модель: Galaxy A55
  [ID: 3] Куртка зимняя — 4800 руб.
     Тип: Одежда | Размер: XL
  ...
Введите ID товара для покупки (0 - завершить): _
```

## Теоретические вопросы

### 1. Что такое геттеры и сеттеры?

**Геттер** — метод, который *читает* приватное поле.  
**Сеттер** — метод, который *записывает* в приватное поле.

Зачем они нужны: поля класса делают `private`, чтобы никто снаружи не мог испортить данные напрямую. Геттеры/сеттеры дают контролируемый доступ.

Пример из проекта — поля `name`, `price`, `balance` закрыты, доступ только через методы:

```cpp
// product.h
class product {
private:
    std::string name;  // закрыто — нельзя менять снаружи
    double price;
    int id;
public:
    std::string getName() const;  // геттер — только читать
    double getPrice() const;      // геттер — только читать
    int getId() const;            // геттер — только читать
};
```

```cpp
// Human.h
class Human {
private:
    double balance;  // баланс закрыт — нельзя просто так прибавить себе денег
public:
    double getBalance() const;  // геттер — можно узнать баланс
    // сеттера нет — баланс меняется только через buyproduct
};
```

Если бы `balance` был `public` — любой код мог бы написать `customer.balance = 999999` и это нарушило бы логику программы.

---

### 2. Что такое полиморфизм простыми словами?

**Полиморфизм** — это когда один и тот же вызов метода ведёт себя по-разному в зависимости от реального типа объекта.

Простая аналогия: ты говоришь каждому животному «скажи звук» — собака лает, кошка мяукает, попугай говорит «привет». Одна команда — разный результат.

Пример из проекта — у всех товаров вызывается `displayInfo()`, но каждый выводит свои данные:

```cpp
// в main.cpp — один цикл для всех типов товаров
for (const auto& item : shopShowcase) {
    item->displayInfo();  // какой displayInfo вызовется — зависит от реального типа
}
```

```cpp
// product::displayInfo() — базовый
void product::displayInfo() const {
    std::cout << "  [ID: " << id << "] " << name << " — " << price << " руб." << std::endl;
}

// electronics::displayInfo() — добавляет модель
void electronics::displayInfo() const {
    product::displayInfo();                              // сначала базовый вывод
    std::cout << "     Тип: Электроника | Модель: " << model << std::endl;
}

// food::displayInfo() — добавляет срок годности
void food::displayInfo() const {
    product::displayInfo();
    std::cout << "     Тип: Еда | Срок годности: " << expirationDate << std::endl;
}
```

В `shopShowcase` лежат указатели типа `product*`, но за ними скрываются `electronics`, `clothes`, `food`. Благодаря ключевому слову `virtual` C++ сам разберётся, какой именно `displayInfo` вызвать.

### 3. Может ли абстрактный класс иметь конструктор?

**Да, может.** Абстрактный класс — это класс, у которого есть хотя бы одна чисто виртуальная функция (`= 0`). Создать *объект* такого класса напрямую нельзя, но конструктор у него быть может — и он вызывается конструктором наследника.

Пример: если бы `product` был абстрактным:

```cpp
class product {
protected:
    std::string name;
    double price;
    int id;
public:
    // конструктор есть — он инициализирует общие поля
    product(std::string& name, double price, int id)
        : name(name), price(price), id(id) {}

    // чисто виртуальная функция — делает класс абстрактным
    virtual void displayInfo() const = 0;

    virtual ~product() = default;
};
```

```cpp
// product p("test", 100, 1);  // ошибка — нельзя создать объект абстрактного класса

// но можно создать наследника — он вызовет конструктор product через список инициализации
electronics::electronics(std::string& name, double price, int id, std::string& model)
    : product(name, price, id), model(model) {}
//    ^^^^^^^^^^^^^^^^^^^^^^^^^^^
//    вот здесь вызывается конструктор абстрактного родителя
```

Итого: **создать объект** абстрактного класса нельзя, но **иметь конструктор** — можно и нужно, чтобы наследники могли инициализировать общие поля через `super`-вызов.

## От себя

главная ошибка которую я допустил изначально — `buyproduct` принимал `cart` как параметр, но **не добавлял** его в `this->cart`. из-за этого `showCart()` всегда показывала пустую корзину. логика была написана, но данные уходили в никуда.

ещё один нюанс: `#include <Windows.h>` определяет макросы `min` и `max`, которые конфликтуют со `std::numeric_limits::max()`. решение — `#define NOMINMAX` перед инклудом, чтобы Windows не трогала эти имена.
